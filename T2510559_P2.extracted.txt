Abstract
The remarkable rise of Large Language Models(LLMs) has made a significant paradigm
shift in the field of Artificial Intelligence. LLMs have shown amazing capabilities in a
wide range of Natural Language Processing (NLP) tasks. However, the computation cost
and memory requirement of the models are high, making them quite resource-intensive.
This is a major obstacle to their deployment in resource-constrained environments or
platforms, such as local servers, mobile devices, IoT devices, or other edge devices. The
existing suite of compression techniques, like knowledge distillation, Low-Rank Adap-
tation(LoRA), pruning, and quantization, has all proven effective in reducing the com-
putational and memory overhead of LLMs. However, most of them remain limited to
static methods, applying a fixed compression level that fails to adapt to dynamic changes
in available memory, latency requirements, or prompt complexity. This study aims to
develop a novel, resource-aware hybrid framework that dynamically selects the optimal
combination of compression techniques during inference. This involves developing a re-
inforcement learning controller agent and integrating it with state-of-the-art pruning
and quantization methods. Afterwards, the dynamic hybrid framework will be evaluated
against static baselines on standard benchmarks, assessing accuracy, inference speed, and
memory footprint. The expected contribution is a practical, adaptive, and sustainable
solution that enables the broader deployment of powerful LLMs on everyday devices.
Keywords: Large Language Models (LLMs), Hybrid Compression, Dynamic
Optimization, Adaptive Systems, Pruning, Quantization, Edge Devices, Resource-
Aware Inference, Adaptive Controller Agent,Reinforcement Learning

Chapter 1
Introduction
1.1 Background
In recent years, Large Language Models (LLMs) have made significant advancements in
the field of artificial intelligence. Open-source LLMs such as BERT, Stable Diffusion,
Llama, etc have created opportunities, including high-end text analytics and code effi-
ciency, to the design of digital art. Such models are constructed on elaborate designs
of billions of parameters, which enable them to comprehend and impressively produce
human-like content with remarkable accuracy. The issue is that such LLMs need im-
mense amounts of computational power and intensive resources. They typically require
advanced processing tools, storage, etc., and high-end memory, which is not affordable to
most people with standard devices. When a person wishes to run an LLM on a weaker
machine, they must manually tweak its settings. This involves a lot of trial and error,
adjusting things like model size through pruning or reducing the precision of its calcula-
tions through quantization. This process is time-consuming and requires a lot of technical
expertise. Due to such difficulties, a widening disparity exists between the capabilities of
these LLMs and the people capable of utilizing them. This weakness hinders the spread
of AI in real-life use and blocks many developers and scientists from being at liberty with
these formidable weapons.
1.2 Rationale of the Study or Motivation
The main motivation behind this research is to break down the hardware barriers that
currently limit the use of LLMs [4]. Right now, the way we deploy these models is
largely static; a model is released with a single configuration that assumes the user has
powerful hardware. This “one-size-fits-all” approach is inefficient and exclusive to spe-
cific hardware and resources. It’s similar to a modern video game being released with
only “Ultra” graphics settings, making it unplayable on most standard computers. In
modern video games, developers solve this by offering a range of graphics settings (low,
medium, high) and often an “auto-detect” feature that configures the game according to
the device the game is being played. This is precisely the kind of solution that LLMs
require. This study proposes to develop an adaptive system that infers resource-aware
auto-detection settings for large language models. The goal is to create a compression
framework that can first analyze the user’s hardware—identifying the type of processing
units, the amount of available memory, storage, and the capabilities of the processing
units—and then automatically adjust the LLM’s parameters for the best possible perfor-
mance on that device. It’s not just about making the model run; it’s about making it
run optimally. This means finding the perfect balance between speed and accuracy for
that specific hardware, ensuring the user gets the best experience their device can offer.
By creating such a system, it can democratize the use of LLMs for all groups of people.
Students could run sophisticated models on their personal laptops for research, devel-
opers in smaller companies could integrate AI features into their mobile apps without

could experiment with LLMs more efficiently. As AI is becoming part of our daily lives,
it needs to be more inclusive and efficient, moving away from a static deployment model
to a dynamic one where powerful tools adapt to the user, not the other way around.
1.3 Problem Statement
The promotional spread of LLM as the component blocks of generative AI, a wider eval-
uation and application is restricted due to the memory and processing demands of LLM,
which are impractical to realize on an individual computer. This is mostly with respect
to the world of limited resources i.e. through personal computers, mobile devices, edge
sensors, local servers etc. which cannot accommodate the high end GPUs. The primary
concern is to take those strong patterns further and their application to the context in
which computationally demanding challenges cannot be performed. The first actual effort
to address this problem has been clouding down to static compression techniques, which
include pruning, quantization, knowledge distillation, and so on. The techniques reduce
the model permanently, including its strength in advance of deployment. Though it is
good to minimise the model footprint, this non-adjustable single size fits all approach, by
definition, is inflexible. It fails to adapt to the dynamic environment which is uncertain in
the true working situations. As an illustration, the memory of a device that is available,
processing power and battery power might differ significantly, and the complexity of user
prompts. Model used is statically compressed and will fail to optimize to such changes
and may fail to operate efficiently and consume too much energy which may later fail
in due course should resources be overstretched. It has led to the need to seek hybrid
compression schemes, which is a combination of methods like pruning and quantization.
The hybrid methods are however not dynamic: before, a compression strategy is fixed,
and the same strategy is applied during the inference. This comes at a tradeoff that does
not entail a major opportunity to maximize. The outstanding critical matter is in no
dynamic frame work to change the degree and type of compression on the fly based on
the temporary availability of resources in the machine, along with those of the subsequent
job. More recent research has never taken into account systems that explicitly integrate
the real-time and resources-aware decision-making process into the compression pipeline.
Such a system contains the elements, among others, of pruning such networks with algo-
rithms, including the popular ones, e.g., Wanda, and quantisation of the weights to fit
into a given number of bits, e.g., GPTQ. A combination of the various elements under a
dynamic control mechanism is what has not been succinctly achieved. This is a large re-
search gap to come up with an intelligent adaptive framework that is capable of selecting
the optimal compression policy in a resource-constrained and dynamic environment. One
such system would then be aggressive pruning and quantization when the battery is low.
The user prompt display would be in a less compressed form when the device is being
charged up and the job being done by the user requires the greatest level of precision.
Therefore, the key point is that the LLMs are still unable to manage themselves when it
comes to using resources based on the ever-changing environmental conditions and given
tasks. A dynamic compression framework is not only a step towards the right direction.
Nevertheless, it is a necessary modification to make sure that LLMs can be carried and
can be made sustainable. It can ensure the expansion of the range of the potent AI in
low-resource contexts, such as NGOs or offline infrastructures, and the possibilities of

that are more powerful but less resource-consuming, to be introduced to the world.
1.4 Objective
Also relying on the current advances in the sphere, this study will work on the new
framework of dynamic, hybrid compression of LLMs, which will be able to adjust to the
alterations of a real-time environment. Prior art in LLM compression is static, i.e., using
a given amount of optimization that does not consider the deployment environment’s
volatile resources, such as mobile devices or edge servers. It is not efficient when it comes
to this so-called one-size-fits-all method. This study closes this essential gap by creating
a system capable of using feedback at the hardware level and the complexity of the user
prompt to change its strategy on compression dynamically.
The central point of this research is to develop a lightweight Reinforcement Learning(RL)-
driven Controller Agent to keep track of the device statistics (CPU load, GPU utilization,
free memory, and battery level). The controller will, in turn, choose and implement the
most suitable set of compression methods. It will select the most appropriate pruning
configuration in real-time choosing components such as attention heads, KV caches, feed-
forward neurons, and transformer layers in response to these dynamic inputs. This will
lead to a temporarily optimal LLM dedicated to the particular task and the present
state of the device, thus making the inference efficient with barely noticeable accuracy
drop. Building on top of existing open-source frameworks and using existing compression
libraries, this study will create empirical insight into the trade-offs of computational
resource consumption and quality of models. The end objective would be to develop a
modular, scalable, and sustainable dynamically hybrid framework system that ensures
that advanced deployment of LLM would be viable and sustainable across a broader
scope of devices, especially in resource-constrained environments. This research will add
a realizable and flexible solution to a prominent issue of the operationalization of large-
scale AI.
Research Objectives of this research include
1. Develop a lightweight, RL-based controller agent that continuously profiles device
telemetry (CPU utilization, memory availability, GPU load) alongside incoming prompt
complexity metrics to inform compression decisions.
2. Create an integrated and flexible hybrid compression module, combining adaptive
structured pruning that integrates attention-head, feed-forward, transformer layer, and
KV-cache pruning with rapid-switch orchestration at inference time.
3. Implement adaptive inference protocols that modulate compression levels based on
resource availability (including memory usage, latency targets, and energy constraints)
within acceptable accuracy bounds.
4. Design and execute comprehensive benchmarking under controlled simulations of ex-
treme device constraints to evaluate end-to-end throughput and quality-of-service.
5. Train and validate the RL agent that will learn optimal pruning strategies balanc-

6. Perform rigorous multi-factor analysis to quantify the frontiers between compression
aggressiveness and model performance metrics, including perplexity, accuracy, inference
speed, and energy consumption.
1.5 Methodology in Brief
Figure 1.1: Methodology Roadmap
The study employs state-of-the-art hybrid model compression techniques, accompanied
by a controlling agent to improve the efficiency of LLMs using an adaptive and dynamic
framework. The research methodology is structured and systematically illustrated in the

the foundational benchmarks to implementing the core dynamic components, concluding
with a comprehensive comparative evaluation. The methodology is as follows:
1.Model Selection:A medium-scale open-source LLM like LLaMA-3(1B) will be se-
lected as the base architecture. They will be chosen based on their open availability,
strong performance, and suitability for running in resource-limited environments.

2.Baseline Static Compression:A standard, state-of-the-art static hybrid com-
pression pipeline will be implemented. In this process, a fixed level of pruning will
be applied to the base model. For example, we will use a one-shot, activation-
aware pruning method like Wanda [27] or SparseGPT [20] to achieve a pre-defined
sparsity (e.g., 50%).
3.Baseline Performance Measurement:The performance of the static hybrid
compression model will be rigorously evaluated on a suite of standard benchmarks,
including reasoning (GSM8k, MMLU) and instruction-following (OpenOrca) tasks.
Key metrics to be recorded will be accuracy, latency of the inference, and peak
memory footprint in the final phase. Currently, we are using our mock dataset to
understand the performance.
4.Controller Agent Implementation:The Reinforcement Learning Controller
agent, regarded as the brain of the framework, will be developed. Designed to re-
ceive real-time context as input, including device statistics (available RAM, CPU
load, etc) and prompt characteristics (e.g, token count, complexity heuristics, perplexity)–
and give an optimal compression policy as output for that specific inference request.
Initially, we will start with a rule-based heuristic controller based on a set of logical
rules, such as checking prompt length and RAM, and then applying the fixed prun-
ing ratios and quant bits for that particular condition. Afterward, we will extend
this by using a Reinforcement Learning (RL) controller agent (e.g, a lightweight
PPO or DQN algorithm), following the paradigm set by frameworks like RAP
(Runtime-Adaptive Pruning for llm inference). The agent will learn a compre-
hensive policy by optimizing a reward function, balancing both inference speed,
accuracy, and resource consumption.
5.Experimental Evaluation:Various resource-constrained environments will be
simulated, which involve programmatically setting budgets for memory usage and
simulating different levels of CPU or GPU load to mimic real-world device con-
ditions. The experiments will be conducted under various simulated constraints
and with a wide range of prompts, ranging from factual queries to long, complex
prompts.
6.Data Analysis:Finally, the performance metrics indicated by accuracy, latency,
and memory usage will be systematically collected. Then, a comparative analy-
sis will be performed, in which our dynamic hybrid framework will be contrasted
against the performance of the static baseline.
1.6 Scopes and Challenges
This research aims to develop a resource-aware hybrid compression model framework for
LLMs, with the capability to dynamically adapt to compression strategies during infer-
ence based on the available system resources. By integrating different pruning techniques,
followed by active monitoring by the controller agent, the system seeks to improve LLM
efficiency without compromising performance. The framework will make LLMs more
adaptable and deployable on resource-constrained devices such as local servers, mobile
phones, and IoT devices. The key challenges and constraints are:

1.Dynamic Compression Policy Selection:It is technically challenging to design
a dynamic hybrid system that adjusts the compression techniques (pruning, quan-
tization) based on real-time conditions (CPU load, memory, input length) without
introducing excessive computational overhead.
2.Accuracy Preservation:There is a possibility that active, vigorous dynamic
compression will lead to a significant decline in the accuracy and performance of
the model. This trade-off requires an appropriate strategic design of the control
policy to be managed.
3.Scalability of the Framework:The framework must be able to scale properly to
use large models and handle various tasks while being resilient when implemented in
real-life situations. The variability of datasets and the challenge of generalizability
make both devices and applications more complex.
4.Accuracy of Simulation of Diverse Resource Environments:The accurate
simulation of the conditions in a resource-sparse environment is rather difficult to
achieve. Simulation should consider dynamic resource states, i.e., variable memory,
CPU usage, or battery power, which may influence performance.
5.Evaluation and Benchmarking the Framework:Exertion in real-world con-
ditions regarding latency, memory utilization, and accuracy ought to be evaluated
with utmost precision to benchmark the framework. The system will have to pass
through a number of standard NLP benchmarks that will enable us to compare the
performance of the static baseline while simultaneously considering the variability
of the resource.

Literature Review
2.1 Preliminaries
Pruninginvolves eliminating unwanted components and those that are of the least sig-
nificance in the model, thus reducing the size, complexity, and overload of a model. That
makes a small model more effective, takes fewer resources, and is faster to use.
Knowledge Distillationis the ability to appropriately train a small model (the stu-
dent) such that it behaves like a massive model (the teacher). The miniaturization will
then mimic the actions of the bigger one with an intensive reduction of the resources
needed. The result is a leaner, more effective model that does not forfeit most of the
performance of the teacher model.
Quantizationrefers to the technique where weights of a model are coarsened (i.e., 32-
bit floating point to 8-bit integer). This reduces the memory usage to a minimal degree,
resulting in speeding up inference. Although this model may lead to reduced precision.
Hybrid compressionattempts to strike some compromise in the model accuracy-size-
speed triad, and employs many techniques of compression pruning and quantization.
Hybrid compression combines the superior capabilities of a broad range of approaches
and can enable models to be compressed by a significant margin, and sped up without
sacrificing performance in a significant way.
Resource-Aware systemsare systems that can dynamically vary their operation as a
reaction to the real-time availability of resources, e.g., battery power, memory, or com-
puting capacity. With respect to LLMs, a resource-aware system might dynamically
determine the compression methods at inference time based on the present system char-
acteristics. It is relevant when operating the models in the real practical environments
that have limited availability of resources, such as cell phones or edge stations.
Reinforcement Learning (RL)is a kind of machine learning whereby an agent learns
to make choices based on the experience it can have upon interaction with its environ-
ment. Depending on its action, the agent will be rewarded or punished. It allows the
agent to make the optimum choice when looking to do something in the end. RL finds
many applications in areas such as decision-making systems, gameplay, or robotics, due
to its capability of making sequential decisions in dynamic environments.

2.2 Review of Existing Research
In a comprehensive survey, Zhu et al. [1]investigated the critical field of model com-
pression techniques in LLMs. The article’s main goal was to provide a comprehensive
overview of existing methods, challenges, and potential directions to improve efficiency
and access to LLMs, especially where computational resources are limited. The authors
avoided embarking on new experimental research; instead, they aggregated evidence from
a broad collection of existing research. These experiments used diversified data sets to
assess compressed models’ effectiveness. The performance of language models was mea-
sured using data sets like WikiText-2, C4, and PTB. For zero-shot task evaluations and
reasoning skills tests, benchmarks like LAMBADA, PIQA, OpenBookQA, and GSM8K
were used. The manuscript further pointed out the usage of large benchmark sets, like
BIG-Bench, and test tools like the EleutherAI LM Harness, to ensure homogenous and
comparable results. The survey covered a broad spectrum of LLMs, including models
from the LLaMA series, OPT, BLOOM, and GPT series. The choice to review these
models stemmed from their widespread use and the significant computational challenges
they present due to their massive size. The paper systematically categorized the com-
pression technologies into four main types: quantization, pruning, knowledge distillation,
and low-rank factorization. It detailed various sub-methods, such as Weight-Only Quan-
tization, Structured Pruning, and Black-box Knowledge Distillation. The performance
of the compression techniques was measured using several metrics. The most promi-
nent metrics were the compression and speedup ratios, representing the models’ reduced
size and improved inference time, respectively. Another notable metric used was the
perplexity difference, which measures the loss in performance during language modeling
after compression. The results demonstrated achievements such as weight-only quanti-
zation methods through GPTQ, which obtained a 3.24x speedup with a slight increase
in perplexity for the OPT-175B model. Similarly, unstructured pruning methods like
SparseGPT achieved a 50% compression ratio with minimal impact on performance.
Thus, the survey effectively presented the considerable progress within the field, demon-
strating that notable gains in efficiency and compression are indeed possible. In short,
this survey forms a substantial and valuable manuscript. It adequately organizes and
describes the existing state of LLM compression, making for a strong foundation for re-
searchers and practitioners working toward more effective language models.
Based on another survey by Bai et al. [2], a wide array of efficiency methods was an-
alyzed across the entire lifecycle of an LLM. These techniques were categorized into
architecture design, pre-training, fine-tuning, inference, and system design, each with
specific reported outcomes. In architecture design, methods focused on optimizing the
core Transformer attention mechanism. Approximate attention techniques aimed to re-
place the standard mechanism, which has a quadratic time complexity ofO(T 2d) for time
andO(T 2) for memory, with more efficient alternatives. For instance, the Reformer model
used locality-sensitive hashing to achieve a near-linear time complexity ofO(TdlogT).
AFT-simple went further, eliminating dot-product operations for a linearO(Td) com-
plexity in both time and memory. Another approach, Modular Networks (MoE), enabled
massive model scaling while maintaining constant computation. The GLaM model, with
1.2 trillion parameters, required half the computational power (FLOPs) for inference com-
pared to GPT-3 by activating only 8% of its parameters for each input token. During the
fine-tuning and inference stages, model compression techniques proved highly effective.

Ma et al. [7] present LLM-Pruner, a framework for task-agnostic compression of large lan-
guage models. One of their main goals was to perform this compression without heavily
relying on the original training dataset, which was frequently proprietary or impractical
to scale. They take a three-step approach to structural pruning. First, the framework
analyzes the model automatically during the ”Discovery” stage. ”Coupled structures” are
groups of neurons and weights that depend on one another. This will guarantee that all
other components that rely on a deleted piece must also be deleted in order to maintain
the model’s integrity. The framework then evaluates the importance of each of these
groups in the ”Estimation” stage by looking at their gradient information, which shows
how they affect the model’s performance. Finally, once the groups of lesser importance are
pruned, a fast”Recovery” phase involves Low-rank Adaptation (LoRA) for fine-tune the
compressed model and reacquire lost performance effectively and efficiently. The authors
showed that LLM-Pruner on various models, LLaMA, and Vicuna, and able to strip 20%
of the parameters, retaining close to a 95% of the performance of the original model after
the careful tuning process with 50,000 data samples and then just in a relatively short
timeframe of three hours. In spite of the novelty of its low resource pruning methodology,
LLM-Pruner is a fixed optimization scheme. The amount of compression is not deter-
mined automatically, rather a developer have to do (specify manually) a fixed pruning
ratio before starting the process. The framework carries out the specified instruction
and produces a single, permanently pruned model designed for the chosen ratio. This
design lacks the ability to adjust compression levels dynamically during operation. For
instance, it cannot assess the memory capacity of a target device and then determine
the most suitable pruning ratio automatically. As a result, when developers need to de-
liver models to devices with varying hardware capabilities, they must repeat the pruning
process manually for each ratio, which can be labor-intensive and inefficient. Moreover,
the authors point out that higher pruning levels,such as 50%,can cause remarkable drops
in performance, underscoring the limitations of a fixed-ratio method. Because the com-
pression level is set in advance and not adapted in real time, the model produced by
the framework is locked to the specified ratio. This static nature means that the system
cannot fine-tune compression based on live resource constraints, making it impractical for
flexible, multi-device deployments. High pruning rates can lead to a major fall in model
performance, which highlights the difficulty and rigidity of a fixed-ratio approach.

Chen et al. [24] presented DLP, a dynamic layerwise pruning algorithm for accelerat-
ing LLM inference, aimed at reducing compute latency and memory on the fly with no
sacrifice in output quality. Its core goal was to enable at runtime choosing transformer
layers to skip based on input complexity and target latency, avoiding the inflexibility of
static pruning and the expense in generalization over tasks or prompts. To evaluate DLP,
the authors utilized both reasoning and generative tests. Language model testing was
carried out using WikiText-2, and downstream testing was done with MMLU and Open-
BookQA. Tested models included LLaMA-7B, OPT-6.7B, and Mistral-7B, which offered
ample testing on dense and decoder-style models. Inputs were preprocessed into fixed
prompt templates that included mixed instruction-following and reasoning tasks. Train-
ing included standard autoregressive loss, sparsity scheduling, and pruning fine-tuning.
The method introduced an importance score layer in terms of the activation norm as
a signal to tell which transformer blocks to prune for a given input. DLP then used a
scaling scheduler to scale between target latencies and pruning ratios dynamically during
inference. Importantly, no retraining from scratch or access to external gating predictors
is required. Fine-tuning was used to achieve stabilization of the pruned model for con-
sistent performance. For performance measurements, DLP was quantified as inference
latency, perplexity, F1/Accuracy, and token throughput. For LLaMA-7B, DLP achieved
2.1×speedup and 35% lower latency at a ¡1% loss of accuracy on QA tasks. Perplex-
ity loss was minimal on WikiText-2 (from 5.4 to 5.6), much less than static pruning
baselines like LayerDrop and uniform layer skipping. DLP was also shown to generalize
strongly to zero-shot tasks without additional tuning. Even though successful, DLP did
not include quantization or memory profiling in decision-making logic. Pruning was lim-
ited to whole layers, and more detailed optimizations like attention head or MLP block
pruning were not investigated. More extension of DLP to multi-objective compression,
hardware-specific scheduling, and hybrid architecture was suggested by the authors. This
study directly validates our research direction, illustrating that runtime-aware pruning
enhances LLM inference speed. DLP’s simplicity in architecture, strong performance,
and capacity to generalize across tasks validate dynamic, structured compression within
hybrid optimization frameworks.

With particular focus on facilitating a faster inference process, Lin et al. [4]developed a
powerful inference engine called ScheInfer to accelerate the process of inference of LLM
on single, moderate GPU-enabled systems. The key purpose of the system is to get some
knowledge of the type getting performance better without changing the accuracy of the
model. Its key strategy is a novel model partitioning method composed of the following
three parts to be done in parallel: 1) a CC (CPU-Compute) part which is stored and
computed on the CPU; 2) a CG (CPU-GPU) part, which is stored on the CPU in order
to save GPU memory, but conveyed to the GPU to be computed; and 3) a GG (GPU-
GPU) part, which is stored and computed on the GPU only. By organizing such pipeline
operations in an asynchronous fashion, ScheInfer attempts to overlap usage of the CPU,
GPU and PCIe communication resources of the system whenever and to the greatest
extent possible. Perhaps the most important part of this system is an adaptive parti-
tion algorithm; a ”Profiler & Solver” module that first constructs a performance model
of the target hardware and then computes the best ”slicing rates” of the CC, CG and
GG partitions that will minimise the inference latency. Experiments revealed that the
approach is capable of offering large end-to-end inference speedups, with factor of 1.25x
to 2.04x compared to the state of the art implementations, including, e.g., llama.cpp.
While ScheInfer proposes for an enhanced way of resource management, the optimization
approach is static and is done on the scheduling level which is not the model level. The
system generates an optimal method of running a static model (one that is given) on a
specific hardware configuration. The profiling and solving takes place once beforehand
for finding out the best way to divide up the workload. It doesn’t make any model com-
pression (like pruning and quantization), and impacts nothing the features of an initial
model. Thus system is not adaptive as hyper paras or architecture of the model is not
changed to suit the available hardware. It does give an intelligent schedule in running a
static model more efficiently according to static resource constraints, but doesn’t address
the question whether a smaller or less complex version of the model would be inherently
better suited to the resource constraints - demonstrating the need for a system with the
capability of adapting the model itself.

Wang et. al [23] introduced DJPQ, a new compression framework that optimizes both

proximation of hardware efficiency. Optimization parameters on both sparsity and bit-
width enabled end-to-end training in a single optimization pass, in contrast to iterative

2.3 Summary of Key Findings
This research aims to solve one of the most significant issues that prevent the mass use
of LLMs: the inability of modern compression algorithms. Albeit effectively reducing
LLMs’ huge CPU and memory demands, pruning, quantization, and knowledge distilla-
tion are primarily static, one-time solutions. This generic approach thus ends up with
de facto models locked away in a permanently compressed state and never allowed to
respond dynamically to changing environmental demands under which practical systems
have to operate, e.g., memory availability, CPU availability, battery state, or complexity
of user request. Such an intrinsic lack of flexibility greatly limits the implementation
of large, sophisticated LLMs on devices with limited resources, e.g., cell phones, local
servers, and the Internet of Things (IoT) devices. Due to a significant gap in doctrines,
the present study suggests developing a new resource-efficient hybrid model that selec-
tively and methodically decides on the most desirable balance between the compression
methods during inference. Second, the most notable is a better Controller Agent devel-
oped with great consideration for real-time operation. The controller not only reads the
telemetry of the devices, e.g., CPU consumption, memory size, battery level, etc., but
also estimates the quality of the prompts. Based on this real-time data, the controller will
go into an optimal compression strategy, which will be obtained by dynamically varying
the pruning and quantization rates correlated with the particular inference request. This
kind of modern technique is comparable to the action of modern video games when they
detect and adjust the parameters automatically accordingly to achieve the best perfor-
mance based on the hardware on which the system is being used by the user, which
eases the favoring of the system whereby, the AI can adapt to the user instead of the
reverse being true. The track in which the present paper is written is system-focused
towards developing and testing the target system. Phase one would choose a strong,
de-restricted, open-source foundation model, e.g., Llama 3 or Phi-3. They will present
a bottom-line performance metric, implemented using a fixed hybrid compression path
using state-of-the-art methods, i.e., Wanda to prune the models and Activation-aware
Weight Quantization (A WQ) to quantify them. Next, a dynamic system will be built,
with a dynamic controller consisting of a rule-based but increasingly more advanced
RL learner capable of learning complex optimizing approaches. It is necessary to test
comprehensively strong prior understandings and instruction-execution rules to measure

accuracy, delay of inferences, and memory usage with constraints of virtual hardware.
It is noted that the overall problematic technical issues that the current study gallops

model accuracy even subject to thorough compression, as well as the anticipation of the

unified intelligent control system, hence creating an efficient and feasible solution to scale
large, high-performance language models on standard hardware.

complexity score is statistically sound, directionally consistent, and interpretable across
contextual and linguistic factors.

prompts show systematically higher complexity scores, category-level distinctions are

with assigned difficulty. Together, these findings demonstrate that the dataset is inter-

system-level dataset validation. While these results confirm internal and construct va-

Rationale for Omitting Explicit Dataset Preprocessing
For this phase of the research, the dataset we used was primarily manually curated prompt
collections that were designed to evaluate the performance of existing pruning and com-
pression frameworks. The mock dataset that we created was cautiously constructed,
covering prompt-level complexity scoring and basic cleaning like standardization of cate-
gorical labels and removal of duplicate prompts. Contrary to common machine learning
workflows, this level of data curation does not consist of extensive preprocessing.
The pruning frameworks evaluated, such as LLMPruner, SliceGPT, and SparseGPT,
operate directly on tokenized inputs and model weights, relying on small calibration
slices derived from held-out evaluation splits. These methods do not mandate additional
data normalization, augmentation, or feature extraction steps prior to execution. Their
core inputs are token sequences and associated model parameters, and calibration data
serves for importance estimation rather than requiring processed features.
Adding to that, in our proposed framework, the reinforcement learning-based controller
agent also utilizes prompt metadata like token length and complexity scores encoded in
state vectors, alongside device telemetry. This information is aggregated or extracted but
does not essentially require preprocessed raw data for the pruning decisions.
Therefore, explicit dataset preprocessing was not a prerequisite for running the existing
pruning frameworks or the proposed RL framework. The emphasis was on consistent
dataset design, proper splitting during training of RL agent, and annotation to facili-

tate fair evaluation rather than transforming data for model consumption. Introducing
complex preprocessing stages was neither necessary nor aligned with the goals of runtime
adaptive model compression and inference optimization targeted in this study.
LLM Model Selection
For the existing framework evaluations and our proposed framework, we have chosen the
Llama 3.2 model with 1B parameters since it is the only model that offers both open
availability and has an interesting trade-off in terms of resource efficiency and language
modeling performance. The Llama family of models introduced by Touvron et al. [32] is
designed to provide high performance at different parameter scales and is optimized for
deployment in resource-constrained settings, such as settings with limited GPU VRAM.
We selected the 1B version of Llama 3.2 for the reasons below:
Resource limitations: One of the main driving factors behind using Llama 3.2 1B was
its right balance between computational efficiency and its ability to understand mod-
ern languages. As Touvron et al. [32] stated, the Llama architectures are designed in
such a way that they are optimized in particular for high-quality text generation in a
low-parameter setting. 1B parameter scale is specially suitable for experimentation in
consumer-grade hardware. Having limited resources to run higher parameter LLMs, we
had to opt for Llama 3.2 1B because we had hardwares RTX 3070/4060 with 8GB VRAM
available. Based on the framework evaluations and the proposed framework we had to
run, it would be too expensive and time-consuming to train or prune models that have
tens of billions of parameters. Llama 3.2 1B was feasible for us with our current specifi-
cations, and given its proven balance between accuracy and efficiency for NLP tasks, it
enables iterative experimentation and fast prototyping under hardware constraints.
Model Sufficiency and Suitability to Pruning: The architecture of Llama 3.2 1B,
which includes a transformer base, rotary positional embeddings, and grouped query
attention, is highly efficient. This design not only improves computational speed but
also makes the model well-suited for our planned pruning and dynamic compression
techniques. Its structure aligns perfectly with our goal of using reinforcement learning to
optimize inference.
Advanced Training and Key Features: Llama 3.2 1B benefits from modern train-
ing techniques like knowledge distillation and instruction tuning. It inherits significant
linguistic and generalization capabilities from its larger ”teacher” models (the 70B and
7B versions of Llama 3.1). Furthermore, its ability to handle long input sequences (up
to 128k tokens) is remarkable. Moreover, it is said to maintain a strong multilingual
performance. As a result, these features are essential in effectively managing the varying
and complex prompts during our research.
Most of the existing research we have reviewed employed the Llama models. There are
other models like BERT as Devlin et al. [33] mentioned. BERT also has variants. They
were not chosen in spite of their efficiency in representation and classification tasks. This
is because their underlying architecture is the bi-directional encoder architecture, which
is mainly optimized for non-generative tasks. Since BERT is not natively able to do open-
ended text completion, none of the modern advances in RL and instruction tuning are
found in it, unlike newer generative models. This results in it being a poor architecture
for experimentation in pruning and generation or instruction following. Moreover, ML
models like BERT are less suitable for structured sparsity studies and less granular, and

4.1.6 Baseline Pruner Evaluation
To understand how pruning patterns and light recovery affect language modeling qual-
ity at a small scale and under consumer-GPU constraints, we performed a systematic
evaluation of post-training pruning methods using our dataset from existing works. Our
objective was to ensure the pruning methods were compatible and tweaked enough for
our experimental setup, and to produce results based on our data consistently.
Candidates for Pruning
•SparseGPT:[20] A one-shot, layer-wise pruning method that reconstructs pruned
weights using fast approximate second-order updates; supports unstructured spar-
sity and semi-structured 2:4 patterns, enabling comparisons between accuracy preser-
vation and potential hardware acceleration.
•LLM-Pruner:[7] A structured, dependency-aware group pruning approach that
removes channel/head blocks while maintaining architectural validity; commonly
paired with brief LoRA recovery to restore quality with minimal additional data,
aligning with task-agnostic compression goals. [20]
•SliceGPT:[28] A block-wise orthogonal reparameterization technique that deletes
rows/columns along low-energy directions to shrink dense dimensions directly; promises
realized compute savings, but is sensitive to PCA precision and calibration coverage,
especially for smaller models.
Model targetLlama-3.2-1B was selected to mirror resource-constrained deployment
and highlight method robustness at small parameter counts where representational re-
dundancy is limited, making pruning trade-offs more visible.
Experimental setup
•Hardware and Environment:Experiments were conducted on a Ryzen 7 3700X
CPU, NVIDIA RTX 3070 (8 GB), and 16 GB of RAM, utilizing mixed precision
where supported and default dense kernels when specialized sparse kernels were
unavailable.
•Model:All runs used Llama-3.2-1B without task-specific fine-tuning before prun-
ing; precision and maximum sequence length were held consistent within each
method’s feasible memory envelope to ensure fair perplexity scoring.
•Dataset:Perplexity was computed on a held-out split of a mixed-domain P2 eval-
uation set covering knowledge, commonsense, dialogue, math, and reading com-
prehension, ensuring diverse token distributions; the same text was used across
baselines and pruned variants.
Implementation details
•SparseGPT[20] is executed one-shot and layer-wise, reconstructing weights to
minimize output error per block while pruning both attention and MLP matrices.
Two configurations are used to reflect accuracy-versus-structure trade-offs: 50% un-
structured sparsity and a 2:4 semi-structured mask applied uniformly across layers.

FP16 forward, where stable, and FP32 for sensitive operations, without enabling

•LLM-Pruner[7] follows a structured, dependency-aware strategy that removes

4.2 Preliminary Design and Specification
Our model is based on the Llama-3.2-1B transformer architecture, which is a state-
of-the-art large-scale language model that is very efficient in several natural language
understanding and generation problems as said by Dubey et al. [38]. The selection
of the base model (Llama-3.2-1B) is an important decision informed by an appropriate
balance between representational power and computational efficiency, given that it is the
most suitable model to conduct research on the topic of dynamic pruning. Another reason
was resource constraints. We did not possess enough GPU computing power to conduct
an academic standard 7B or 13B model, but we will conduct that in our future work.
The architecture of a typical transformer proposed by Vaswani et al. [39] is used to
achieve this parameter model (1 billion), which consists of multi-head self-attention and
feed-forward networks in the form of layers. We exploit the modularity of transformer
architectures to perform structural changes during inference in real-time. The natural
targets of pruning are attention heads and the transformer layers of the model, which
43

===== PAGE 54 =====
are easy to control to modify the computational complexity of the model across granular
scales without impacting the underlying representational power of the model. The design
philosophy is consistent with the latest findings in structural pruning studies, indicating
that the various components of a transformer have varying amounts of redundancy and
can be removed without a significant drop in performance [37], [7]. This rationale
supports the choice of structured pruning, which is a faster alternative. Since structured
pruning reduces the dimensions of GEMM operations by removing entire transformer
layers or the attention heads, it leads to predictable and hardware-friendly reductions
in computation and memory usage contrary to unstructured sparsity approaches that
remove individual weights and need sparse-tensor kernels. The architectural foundation
can be utilized by our reinforcement learning-based pruning controller to make dynamic
decisions about which parts to prune based on real-time performance feedback. This
represents a change in paradigm from the more traditional types of pruning, which apply
the same degree of compression to all possible inputs, to an adaptive system that applies a
different form of computation depending on the properties of the specific input sequence.
4.2.1 Custom Architecture Overview
System Architecture
The structure of our adaptive pruning system consists of three main parts: namely, the
base transformer model, the reinforcement learning controller, and the dynamic pruning
engine. The architecture deploys a closed-loop feedback system in which the RL agent
monitors model performance metrics and makes pruning decisions that directly affect
future inference speed and quality. Contrary to the fixed heuristic pruning strategies
that involve thresholds or static rules, a reinforcement learning controller can adapt in
real time to both device load and prompt difficulty. This enables consistent performance
across many different kinds of hardware conditions and varying complexities, consistent
with the findings of Mnihet al.[36].
Base Model Architecture:We are based on the Llama-3.2-1B model, which has 16
transformer layers, each containing 16 attention heads and an intermediate dimension
of 8,192 in the feed-forward networks. Input sequences are handled in this model by an
embedding and then by the transformer stack, with the final stage being the language
modeling head, which produces the probability distribution over the vocabulary.
Reinforcement Learning Controller:A Double Deep Q-Network (DDQN) agent [37]
controls the decisions made by the pruning process and represents an important im-
provement over classical methods of Q-learning. To overcome the overestimation bias
that is characteristic of standard DQN implementations, the DDQN architecture uses
two independent neural networks: a main one(Policy Network) to select actions and a
target one(Target Network) to estimate values [36]. Policy Network Handlesε-greedy
exploration/exploitation, selects actions, and is updated during training. Whereas Tar-
get Network provides stable Q-value targets that are updated periodically to reduce bias.
The selection of Double DQN over policy-gradient alternatives was made to align the
compact pruning space with DDQN’s off-policy nature and discrete action formulation.
As established by Van Hasseltet al.[37], replay buffer-based learning of DDQN pro-
vides higher stability and efficiency. The RL agent state representation includes three
important aspects: input sequence length (between 1 and 50 tokens), prompt perplexity
(calculated on the input sequence), and complexity score (calculated based on dataset an-

Dynamic Pruning Engine
The pruning engine implements real-time structural modifications to the transformer
model during inference. The system employs two primary pruning strategies:
•Attention Head Pruning:Following the methodology in the NIPS [29], our
system selectively removes attention heads based on learned importance scores.
The pruning process involves:
1. Computing attention head importance using gradient-based metrics
2. Applying structured pruning masks to remove entire attention heads
3. Maintaining tensor contiguity for optimal GPU memory access patterns

•Transformer Layer Skipping:Our implementation enables the dynamic by-

inference speedup for certain input types.
Acceleration Layers
To further enhance the effectiveness of real-world inference, our implementation includes

memory bandwidth and latency during autoregressive decoding, we enable graph-level
compilation. These complementary optimizations ensure that pruning-based gains are
effectively translated into observable throughput gains on modern hardware [24].

Chapter 5
Conclusion
The deployment of LLMs on ubiquitous, resource-constrained environments remains of
the most significant and pressing challenges in the field of artificial intelligence. This
research aims to tackle this challenge by trying to optimize the performance and efficiency
of LLMs through proposing a dynamic, hybrid framework that makes LLMs resource-
aware. Using hybrid compression techniques like pruning and quantization, a lightweight
controller agent with the help of Reinforcement Learning will intelligently select pruning
and quantization levels based on real-time device statistics and prompt complexity. This
research looks to go beyond the static compression techniques that already exist. The
remarkable advantage of this approach is that it can try to find the optimal balance
between performance and efficiency at any given moment. This is a flexibility that static
methods do not have. Yet, the journey requires overcoming challenges related to the
controller’s overhead and ensuring accuracy is preserved under aggressive compression.
In short, this research goes beyond a purely technical contribution because it leads to
a future where powerful AI is more accessible, resource-aware, and sustainable. This
research plans to redesign LLM deployment by connecting hybrid model compression
with resource-aware intelligence to dynamically optimize the efficiency of LLMs.
